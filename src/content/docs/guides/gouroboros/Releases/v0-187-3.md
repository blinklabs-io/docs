## v0.187.3 (July 7, 2026)

**Title:** Tolerate duplicate Conway redeemer keys

**Date:** July 7, 2026

**Version:** v0.187.3

This release improves Conway redeemer decoding so blocks with duplicate `(tag,index)` keys sync successfully while preserving hash safety.

### 💪 Improvements

* Fixed **Conway redeemer decoding for duplicate keys:** `ConwayRedeemers.UnmarshalCBOR` now retries lenient decoding after a strict duplicate-map-key failure, so Conway blocks that cardano-node accepts no longer stall sync. Hashes still come from the original CBOR bytes, which preserves hash safety.

## v0.187.3 (July 7, 2026)

**Title:** Keep block syncing moving with duplicate Conway redeemer keys

**Date:** July 7, 2026

**Version:** v0.187.3

This release keeps block syncing moving when duplicate `(tag, index)` redeemer keys appear in block data accepted by `cardano-node`, while preserving redeemer hashes from the original CBOR bytes.

### 💪 Improvements

* Fixed **Conway redeemer decoding for duplicate keys:** `ConwayRedeemers.UnmarshalCBOR` now retries lenient decoding only after a strict duplicate-map-key failure, so Conway blocks that cardano-node accepts no longer stall sync. Hashes still come from the original CBOR bytes, which keeps hash behavior unchanged.

## v0.187.3 (July 7, 2026)

**Title:** Keep block syncing moving with duplicate Conway redeemer keys

**Date:** July 7, 2026

**Version:** v0.187.3

This release keeps block sync moving when duplicate `(tag, index)` redeemer keys appear in block data accepted by `cardano-node`, and it preserves redeemer hashes from the original CBOR bytes.

### 💪 Improvements

* Improved **Conway redeemer duplicate key handling:** Duplicate map key errors now trigger lenient decoding, so block sync keeps moving when `cardano-node` accepts block data with repeated `(tag, index)` keys. Redeemer hashes still come from the original CBOR bytes.

Thank you for following gOuroboros releases.

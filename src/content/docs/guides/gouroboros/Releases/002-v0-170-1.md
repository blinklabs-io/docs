## v0.170.1 (May 15, 2026)

**Title:** Validation hardening, protocol safety, and behavior clarifications

**Date:** May 15, 2026

**Version:** v0.170.1

Hi folks! Here’s what landed in v0.170.1.

### ✨ What's New

* Noted **no new features:** This patch release focuses on improvements, fixes, and release note maintenance.

### 💪 Improvements

* Strengthened **reward withdrawal witness checks:** Reward withdrawals that use key credentials now require the matching witness, which helps validators reject withdrawals that are not properly authorized.
* Refined **block local verification guidance:** `VerifyBlock` and `ValidateStage` check block level rules, but they do not act as full chain context consensus gates, which makes their validation scope clearer for integrators.
* Sharpened **pipeline drain visibility:** Pipeline draining now counts work that is still applying, which gives callers a more accurate view of pending block processing during shutdown and drain flows.

### 🔧 Fixes

* Corrected **VRF key hash validation:** Header validation now checks the canonical VRF key hash format so pool registrations that use legacy hash widths fail clearly instead of slipping through with the wrong key hash shape.
* Hardened **duplicate map key handling:** `CBOR` decoding now rejects duplicate map keys so malformed inputs fail early with a clear error instead of producing ambiguous decoded data.
* Closed **unsupported Plutus phase 2 paths with an error:** Transactions that require phase 2 execution in unsupported era paths now fail with an explicit error when marked valid, which prevents false positive validation results.

### 📋 What You Need to Know

* Clarified **patch release scope:** This patch release focuses on validation hardening, protocol safety, and behavior clarifications rather than new features.
* Documented **regression coverage for collateral return behavior:** Regression coverage now protects the recently corrected collateral return behavior so future changes are more likely to catch a repeat of that failure.
* Recorded **release note maintenance context:** The prior `v0.170.0` release note publication change updates release documentation history and does not change runtime behavior.

### 🙏 Thank You

Thank you for following gOuroboros releases.

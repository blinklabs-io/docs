## v0.186.2 (July 5, 2026)

**Title:** Improve legacy pre-Conway compatibility and keep Conway decoding strict

**Date:** July 5, 2026

**Version:** v0.186.2

This release improves compatibility with older pre-Conway data while keeping Conway and Dijkstra decoding strict for malformed payloads.

### ✨ What's New

* Added **duplicate-member checks for Conway and Dijkstra tagged data:** gOuroboros now rejects duplicate members in the covered witness, body, sub-transaction, and guard sets, so malformed tagged payloads fail fast during decode.

### 💪 Improvements

* Improved **legacy pre-Conway MultiAsset decoding:** gOuroboros now accepts duplicate map keys in pre-Conway MultiAsset CBOR with last-wins behavior, which improves compatibility with older chain data during sync and decode workflows.

* Kept **Conway and Dijkstra transaction decoding strict:** gOuroboros still rejects duplicate MultiAsset keys after decode, so malformed newer transactions continue to fail instead of producing ambiguous state.

Thank you for following gOuroboros releases.

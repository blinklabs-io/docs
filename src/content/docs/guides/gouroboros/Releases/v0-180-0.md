## v0.180.0 (June 2, 2026)

**Title:** Add Dijkstra era support and improve Leios reliability

**Date:** June 2, 2026

**Version:** v0.180.0

Hi folks! Here’s what landed in v0.180.0.

### ✨ What's New

* Added **initial Dijkstra era support:** gOuroboros now works with the newest supported ledger era.

### 💪 Improvements

* Improved **Leios vote and certificate handling:** Stake based committees now handle `CIP-0164` votes and endorser block certificates more reliably.

* Stabilized **Dijkstra era stake and parameter handling:** Dijkstra related settings now remain consistent during startup and updates.

* Fixed **low reward pool calculations:** Rewards now remain accurate when a pool’s total rewards fall below the fixed fee.

* Hardened **BlockFetch configuration errors:** Invalid configuration now returns clear errors instead of panicking.

### 📋 What You Need to Know

* Added **routine maintenance updates:** This release covers reward formatting cleanup, minor ledger test cleanup, release notes maintenance, and the `plutigo` update from `v0.1.13` to `v0.1.14`.

### 🙏 Thank You

Thank you for following gOuroboros releases.

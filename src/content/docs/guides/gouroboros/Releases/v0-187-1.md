## v0.187.1 (July 7, 2026)

**Title:** Strengthen genesis staking pool parsing and align Dijkstra block handling

**Date:** July 7, 2026

**Version:** v0.187.1

This release improves genesis staking pool parsing and aligns Dijkstra and prototype-2026w27 block handling with the ledger specification.

### 💪 Improvements

* Fixed **genesis staking pool aliases:** the parser now accepts Shelley genesis names `publicKey`, `vrf`, `owners`, and `metadata` alongside the existing on-chain and API field names, which keeps the VRF key hash attached to genesis-declared pools.

* Matched **Dijkstra and prototype-2026w27 wire handling:** encoding, decoding, and verification now follow the IntersectMBO/cardano-ledger CDDL exactly, including inline transactions, `invalid_transactions`, `leios_certificate`, `peras_certificate`, body-hash verification, and offset extraction.

* Updated **the release notes history:** v0.186.3 now includes the prior release note entry, keeping the published release record complete.

Thank you for following gOuroboros releases.

## v0.183.0 (June 12, 2026)

**Title:** Account state queries, era model cleanup, and safer pipeline validation

**Date:** June 12, 2026

**Version:** v0.183.0

Hi folks! Here’s what landed in v0.183.0.

This release adds the new Conway-era LocalStateQuery GetAccountState API, simplifies ledger era handling by treating Leios as Dijkstra-era overlay functionality instead of a standalone era, fixes pipeline validation so unvalidated blocks are no longer applied when validation is enabled, and keeps the published release history current.

### ✨ What's New

* Added **the LocalStateQuery GetAccountState query for Conway account state:** gOuroboros now returns Conway account state through a dedicated query, making account level state easier to request and use.

### 💪 Improvements

* Simplified **ledger era handling for Leios compatibility:** gOuroboros no longer models Leios as a separate ledger era and now treats it as Dijkstra era overlay functionality, so older Leios imports or flags may need updates to the Dijkstra era model.

* Fixed **pipeline validation so only validated blocks are applied:** gOuroboros now stops blocks that never passed validation from being applied when validation is enabled and returns a clear error instead.

### 📋 What You Need to Know

* Published **the prior v0.182.0 release notes in the release history:** the repository release history now includes the previous version page so the published notes remain current.

### 🙏 Thank You

Thank you for following gOuroboros releases.

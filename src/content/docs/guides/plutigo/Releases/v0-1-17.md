## v0.1.17 (July 14, 2026)

**Title:** Improve dependency reliability and documentation clarity

**Date:** July 14, 2026

**Version:** v0.1.17

This release improves release guidance, refreshes key dependencies, and clarifies reference material so integrations align more closely with current behavior and current maintenance updates.

### ✨ What's New

This patch release focuses on clearer documentation and maintenance updates that keep Plutigo current across day to day use and validation workflows.

* Expanded **release history coverage:** The release library now includes a structured v0.1.16 entry, which makes recent security, performance, bug fix, and documentation updates easier to review in one place.
* Clarified **reference examples:** The parsing examples now match the current API shape, and the V4 cost model notes explain that only one placeholder estimate remains.

### 🔧 Fixes

* Improved **Ethereum dependency maintenance:** Plutigo now uses the latest `go-ethereum` maintenance release, which brings in current upstream fixes and reliability improvements.
* Strengthened **cryptography validation:** The updated CIRCL dependency improves validation hardening and rejects problematic inputs more consistently.
* Refreshed **build validation tooling:** Continuous integration now uses a newer Go setup action across benchmark, test, lint, and nilaway workflows, which keeps automated validation aligned with current toolchain setup.

### 📋 What You Need to Know

This update keeps reference material and dependency maintenance current. Teams that rely on the documented parsing examples or want the latest maintained dependency set can treat this release as the current baseline.

### 🙏 Thank You

Thank you for using Plutigo.

## v0.43.0 (June 25, 2026)

**Title:** Tray notification rules, signed installers, API migration, and more reliable releases

**Date:** June 25, 2026

**Version:** v0.43.0

This release introduces a major tray notification overhaul, broader multi-target monitoring, a signed and notarized macOS `.pkg` installer, more reliable Windows signing with Java 21, a `net/http` API server migration, stronger configuration precedence, and more dependable release and CI workflows.

### ✨ Highlights

* Improved **monitor multiple targets from the tray at once:** Adder now supports wallets, DReps, pools, assets, policies, and monitor everything, batches notifications at a controlled rate, lets connection alerts bypass the limiter, and migrates older tray filter settings into the newer layout.

* Added **ship a signed and notarized macOS `.pkg` installer:** Adder now bundles the CLI and tray app into `Adder.app`, signs, notarizes, and staples the installer when credentials are available, and publishes verified package artifacts with preinstall and postinstall scripts.

* Fixed **sign Windows installers with Java 21:** Adder now uses Java 21 in the Windows signing workflow, which prevents the signing failures seen with Java 17 and keeps signed installer builds moving.

* Refined **run the API server on `net/http` without changing the documented routes:** Adder now uses structured access logging, panic recovery, streaming event handling in `/events`, updated FCM and QR routes, adjusted Swagger paths, and request timeouts that match the live service.

* Clarified **apply configuration values in the right order:** Adder now preserves CLI flags, applies `CLI > YAML > env`, lets YAML override environment variables, and keeps configuration behavior predictable across startup paths.

* Strengthened **keep release publishing moving when one platform fails:** Adder now makes macOS signing non-blocking, pins the macOS runner to `macos-15`, separates app bundle packaging from signing and notarization, and continues final release publication even when some matrix jobs fail.

### 💪 Improvements

* Hardened **save configuration atomically under concurrent writes:** Adder now serializes saves per path, writes to unique temp files, closes the encoder before sync, uses `F_FULLFSYNC` on macOS, syncs the parent directory when supported, and cleans up temp files.

* Updated **decode newer Cardano data more reliably:** Adder now tracks a newer `gouroboros` release with more tolerant Conway redeemer decoding and earlier ledger and decoding fixes.

* Improved **keep transaction evaluation and cryptography current:** Adder now uses newer `plutigo` and `x/crypto` releases that strengthen decoding and evaluation while also adding security hardening and performance improvements.

* Expanded **render bundled images and fonts more safely:** Adder now uses a newer `golang.org/x/image` release with stricter decoding and stronger resilience for malformed visual assets.

* Updated **keep operating system and runtime support current:** Adder now tracks a newer `golang.org/x/sys` release that brings build compatibility updates and upstream runtime fixes.

* Refined **keep operating system and runtime support current again:** Adder now tracks an even newer `golang.org/x/sys` release that carries additional runtime fixes and compatibility updates.

### 🧪 Testing, CI, Docs, and Release Tooling

* Polished **keep lint checks current:** Adder now uses a newer `golangci-lint` action in the lint workflow, which keeps automated checks current.

* Updated **keep Windows signing workflows aligned with the current Java toolchain:** Adder now uses a newer `actions/setup-java` release in the Windows signing step, which keeps the signing environment current.

* Improved **keep Go workflows on the current runner setup:** Adder now uses a newer `actions/setup-go` release across Go workflows while leaving the configured Go versions unchanged.

* Hardened **keep repository checkout behavior current across workflows:** Adder now uses a newer `actions/checkout` release in multiple workflows, which keeps checkout behavior current.

* Refined **keep Docker metadata generation current:** Adder now uses a newer `docker/metadata-action` release in Docker CI and publish workflows.

* Enhanced **keep Maven and Java toolchain handling current:** Adder now uses a newer `actions/setup-java` release in publish and MSI workflows, which improves diagnostics, caching, and toolchain handling.

* Removed **an unnecessary CI step:** Adder no longer runs the issue closed date action during automated checks.

* Streamlined **keep Docker build setup current:** Adder now uses a newer `docker/setup-buildx-action` release in Docker CI and publish workflows.

* Modernized **publish container images with current login tooling:** Adder now uses a newer `docker/login-action` release in the publish workflow.

* Improved **verify release artifacts more reliably:** Adder now uses a newer `actions/attest` release in the publish workflow, which strengthens attestation handling.

* Updated **tighten checkout behavior after forked pull requests and other triggers:** Adder now uses a newer `actions/checkout` release across workflows, which keeps security handling and checkout fixes current.

* Strengthened **publish MSI test artifacts on supported runners:** Adder now uses a newer `actions/upload-artifact` release in the MSI test workflow, which requires Actions Runner 2.327.1 or later on self-hosted runners.

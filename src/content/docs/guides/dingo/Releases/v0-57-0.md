## v0.57.0 (June 24, 2026)

**Title:** Expand MidnightState service support, improve governance recovery, and keep Leios sync moving reliably

**Date:** June 24, 2026

**Version:** v0.57.0

Hi folks! Here’s what we shipped in v0.57.0.

### Recommended Network Compatibility ⚠️

| Network             | Compatible |
|---------------------|------------|
| mainnet             | ⛔         |
| preprod-testnet     | ⛔         |
| preview-testnet     | ✅         |

### ✨ Highlights

* Expanded **run MidnightState as a managed service:** Dingo now runs MidnightState with built in health reporting, service discovery support, optional secure connections, and a bounded shutdown path, which makes service operation easier to manage.

* Enhanced **serve complete forged Leios endorsement data to peers:** Dingo now serves complete forged Leios endorsement payloads through leios fetch, which helps connected peers retrieve the full data they need.

### 💪 Improvements

* Improved **restore governance voting state after snapshot recovery:** Dingo now reconstructs stake pool reward account voting state after snapshot recovery, which keeps governance auto vote behavior consistent after a restore.

* Refined **reduce repeated work during governance proposal tallies:** Dingo now reuses per epoch proposal tallies, which improves tally performance and makes slower tally activity easier to spot.

### 🔧 Fixes

* Corrected **keep sync moving past outdated endorsement backlog data:** Dingo now skips waits on outdated endorsement backlog data, which helps sync continue when older endorsement data no longer affects progress.

### 🧪 Testing, CI, Docs, and Release Tooling

* Updated **keep shared Conway test fixtures aligned across tooling:** Dingo now uses shared Conway test fixture generation from the common mock tooling, which keeps test infrastructure aligned across related projects.

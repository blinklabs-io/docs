## v0.49.1 (May 28, 2026)

**Title:** Clarify Preview swap guidance, expand Leios coverage, and improve database reliability

**Date:** May 28, 2026

**Version:** v0.49.1

Hi folks! Here’s what we shipped in v0.49.1.

### ✨ What's New

* Added **clarify SQL database guidance:** The database guide now maps the SQL store, common query paths, and storage behavior so database work is easier to understand and maintain.
* Introduced **show Preview swap flows more clearly:** The DingoSwap example now shows how applications can read pool data, manage assets, build swap orders, and connect a browser wallet for SundaeSwap V3 activity on Preview.
* Expanded **serve broader Leios data:** Dingo now delivers merged Leios ranking and endorser data to connected applications, which broadens support for downstream services and testing tools.
* Enabled **apply reward voting rules automatically:** Dingo now applies stake pool reward account delegation semantics automatically when governance voting follows CIP 1694.

### 💪 Improvements

* Improved **keep reward tracking more complete:** Dingo now preserves reward state across runs, which improves continuity for reward tracking and treasury views.
* Enhanced **keep Preview swap setup aligned with current browser tooling:** The DingoSwap example now fits current browser build tools more cleanly, which makes local setup and wallet connection behavior more reliable.
* Updated **keep the runtime foundation steady:** Dingo now aligns build, test, publish, and module tooling on current supported components for more consistent operation and maintenance.

### 🔧 Fixes

* Corrected **stop chain reads when callers cancel work:** Chain iterators now stop promptly when the calling process cancels the request, which makes long running reads easier to control.
* Fixed **keep database state aligned with the selected network:** Database operations now carry the active network setting more consistently, which helps nodes avoid mismatched database behavior.
* Resolved **preserve Postgres epoch summaries during database activity:** Postgres installations now keep epoch summary readiness intact more consistently, which makes snapshot state more dependable.
* Strengthened **retain missing transaction outputs during recovery:** Database imports now keep missing transaction outputs and collateral returns more reliably during gap recovery.
* Refined **return clearer transaction submission failures:** Transaction submission errors now preserve valid rejection details, which makes failure handling more reliable.
* Stabilized **surface chain sync send failures sooner:** Dingo now reports chain sync send failures through connection close events so broken sessions are easier to detect.
* Streamlined **trim retained header history during sync:** Dingo now prunes retained header history during normal sync work, which helps long running nodes stay steadier over time.

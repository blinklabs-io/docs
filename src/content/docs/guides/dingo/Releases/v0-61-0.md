## v0.61.0 (June 30, 2026)

**Title:** Expand Midnight indexing, stabilize Leios networking, and simplify SQLite upgrades

**Date:** June 30, 2026

**Version:** v0.61.0

Hi folks! Here’s what we shipped in v0.61.0.

### Recommended Network Compatibility ⚠️

| Network             | Compatible |
|---------------------|------------|
| mainnet             | ⛔         |
| preprod-testnet     | ⛔         |
| preview-testnet     | ✅         |

### ✨ Highlights

* Expanded **scan more Midnight activity with steadier recovery:** Dingo now indexes governance, Ariadne, and candidate activity for Midnight while keeping rollback aware persistence and more reliable backfill behavior, which helps Midnight operators recover and track network activity more confidently.

### 💪 Improvements

* Unified **run the full Preview example stack more easily:** Dingo now ships one shared Docker Compose stack for Preview Dingo, Gov Lens, Blockfrost Explorer, and Sundae Preview, which makes example deployments easier to start and manage together.

* Refined **keep network protocol support current:** Dingo now uses `gouroboros` v0.186.1, which keeps upstream network handling aligned with current support and helps operators run with newer protocol updates.

* Updated **keep cloud service integrations current:** Dingo now uses a newer `smithy-go` patch release, which helps AWS related integrations stay current and dependable.

### 🔧 Fixes

* Corrected **serve large block ranges more smoothly:** Dingo now applies blockfetch backpressure more effectively, which prevents send queue overload and keeps large range serving more stable.

* Fixed **keep Leios endorser sync moving:** Dingo now handles Leios bitmaps with the correct wire ordering, which prevents endorser block sync and backfill from stalling.

* Simplified **start fresh SQLite databases more reliably:** Dingo now auto migrates the full SQLite schema at startup for new databases, which makes first run setup more dependable.

### 🧪 Testing, CI, Docs, and Release Tooling

* Clarified **keep storage and Midnight setup guidance current:** Technical documentation now reflects current storage plugin build tag behavior and Midnight gRPC support, which makes deployment guidance easier to follow accurately.

* Modernized **keep release verification tooling current:** CI workflows now use newer attestation tooling, which keeps build verification and release automation aligned with current requirements.

* Refreshed **keep the prior release record aligned:** The repository release history now keeps the v0.60.1 release record current, which makes recent release tracking easier to review.

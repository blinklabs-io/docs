## v0.58.0 (June 25, 2026)

**Title:** Improve forged-block safety, Leios catch-up completeness, and governance/database reliability

**Date:** June 25, 2026

**Version:** v0.58.0

Hi folks! Here’s what we shipped in v0.58.0.

### Recommended Network Compatibility ⚠️

| Network             | Compatible |
|---------------------|------------|
| mainnet             | ⛔         |
| preprod-testnet     | ⛔         |
| preview-testnet     | ✅         |

### ✨ Highlights

* Improved **validate forged blocks before adoption and diffusion:** Dingo now checks locally forged blocks before it adopts and shares them, which helps operators catch invalid blocks earlier and improves node safety.

* Expanded **complete Leios catch up with historical endorser blocks:** Dingo now backfills older Leios endorser blocks during catch up, which helps nodes recover more complete history when they sync behind the network.

### 💪 Improvements

* Refined **speed up pool stake snapshot queries:** Dingo now directs pool stake snapshot lookups through the intended live UTxO path, which improves query reliability during stake related processing.

* Enhanced **improve Conway DRep vote tally accuracy:** Dingo now tallies DRep voting power across full stake credentials and follows a better data lookup path, which improves governance reporting accuracy.

### 🔧 Fixes

* Fixed **prevent reward replay collisions during recovery:** Dingo now keeps governance, MIR, and pool reap reward credits distinct during replay, which prevents duplicate reward deltas when nodes recover state.

* Corrected **store nullable genesis UTxO links consistently:** Dingo now records empty genesis UTxO link values consistently during database initialization, which keeps repeated setup runs reliable.

* Restored **recover from single peer governance sync plateaus:** Dingo now reconnects a stalled single peer chain sync session after a cooldown, which helps nodes resume progress when governance updates stop advancing.

## v0.63.0 (July 10, 2026)

**Title:** Expand Midnight data access and strengthen chain validation

**Date:** July 10, 2026

**Version:** v0.63.0

This release expands Midnight data access, completes more Blockfrost response details, improves stake and slot accuracy, and tightens inbound block validation across chain processing.

### Recommended Network Compatibility ⚠️

| Network             | Compatible |
|---------------------|------------|
| mainnet             | ⛔         |
| preprod-testnet     | ⛔         |
| preview-testnet     | ✅         |

### ✨ Highlights

* Expanded Midnight state queries so large unspent output result sets can be read page by page.

* Broadened Midnight data access so governance, network parameters, block, epoch, and state details are available through more queries.

* Filled Blockfrost block responses with aggregate and header details so applications receive more complete block data.

* Completed Blockfrost address and transaction output responses with embedded data and reference details so downstream tools can resolve richer output information.

* Corrected live stake totals in pool queries so reported stake values stay accurate.

* Logged rollback cleanup failures for stored data so operators can spot incomplete cleanup sooner.

* Calculated slots across Byron history so chain position handling remains correct for older blocks.

* Applied epoch boundary effects to pools and governance in Leios so network state stays consistent.

* Restored Dijkstra block creation so node behavior follows the expected chain rules.

* Tightened inbound block checks so Dingo rejects malformed or out of order blocks earlier when validation is enabled.

### 🧪 Testing, Docs, and Tooling

* Strengthened Midnight rollback coverage so no event rollback paths stay verified.

* Documented the previous release across the published language set so the release history stays current.

## v0.50.0 (June 1, 2026)

**Title:** Enable browser-ready APIs, add Gov Lens, and improve Mithril bulk-load resilience

**Date:** June 1, 2026

**Version:** v0.50.0

Hi folks! Here’s what we shipped in v0.50.0.

### ✨ What's New

* Enabled **use Dingo APIs from browser based applications:** Bark, Blockfrost, Mesh, and UTXORPC endpoints now support cross origin requests, which makes browser based integrations easier to run directly.
* Introduced **explore governance activity with Gov Lens:** The new Gov Lens example provides a Postgres backed dashboard for viewing governance data and deployment assets for running it.

### 💪 Improvements

* Improved **keep Mithril bulk loads more resilient:** Dingo now defers metadata query indexes during bulk load work and rebuilds them safely after a restart, which helps large imports continue more reliably.
* Expanded **track Mithril backfill progress more clearly:** Dingo now reports detailed backfill progress metrics and hot path timing counters so operators can follow sync work more closely.
* Streamlined **start Gov Lens only after Mithril sync is ready:** The compose example now waits for Mithril sync automatically, which makes startup behavior more dependable.
* Clarified **choose archive and pruning modes with more confidence:** The documentation now gives broader deployment guidance for archive and pruning setups.

### 🔧 Fixes

* Strengthened **recover Badger databases more safely:** Dingo now handles closed database iterators more cleanly and surfaces recovery diagnostics more clearly.
* Detailed **understand forged block decode failures faster:** Dingo now logs clearer CBOR diagnostics when forged blocks fail during re decode.
* Corrected **keep Plutus validation results accurate:** Dingo now applies budget accounting and replay validation rules more consistently.
* Preserved **defer asset policy queries without losing import protection:** Dingo now separates the asset policy query index from the unique import constraint so bulk loads can defer that query path safely.

### 🧪 Examples, Docs, and Release Tooling

* Refreshed **keep the Dingo Sundae Preview example current:** The example now carries updated npm dependencies for a smoother local setup path.
* Removed **simplify release publishing steps:** The release workflow no longer runs npm package publishing.

### 📦 Dependency Updates

* Updated **adopt blinklabs-io/nview v0.14.0:** Dingo now uses the newer `blinklabs-io/nview` release.
* Advanced **move to blinklabs-io/go 1.26.3-1:** Dingo now uses the newer `blinklabs-io/go` toolchain baseline.
* Upgraded **refresh the AWS SDK dependency:** Dingo now includes the newer AWS SDK release.
* Modernized **refresh the smithy-go dependency:** Dingo now includes the newer `smithy-go` release.
* Raised **refresh the google.golang.org/api dependency:** Dingo now includes the newer `google.golang.org/api` release.
* Aligned **refresh the OTLP HTTP exporter dependency:** Dingo now includes the newer OpenTelemetry OTLP HTTP exporter release.
* Renewed **refresh the stdout trace exporter dependency:** Dingo now includes the newer OpenTelemetry stdout trace exporter release.

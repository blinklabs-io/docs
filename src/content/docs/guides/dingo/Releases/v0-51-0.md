## v0.51.0 (June 7, 2026)

**Title:** Configurable storage tuning, ledger peer snapshots, and Leios-era groundwork

**Date:** June 7, 2026

**Version:** v0.51.0

Hi folks! Here’s what we shipped in v0.51.0.

### 💪 Improvements

* Improved **tune storage compression with finer control:** Dingo now lets operators configure compression settings, including ZSTD compression levels, which makes storage behavior easier to match to different workloads.
* Expanded **check ledger peer snapshots directly:** Dingo now returns ledger peer snapshots through local state queries, which makes current peer data easier to inspect during network operations.
* Refined **speed up backfill writes for API data:** Dingo now uses fixed shape database statements during API backfills, which improves write efficiency when large backfills run.
* Enhanced **select log output and verbosity levels:** Dingo now lets deployments choose text or JSON logs and set log levels, while keeping text as the default unless JSON is configured explicitly.
* Advanced **prepare for Dijkstra and Leios vote support:** Dingo now includes early groundwork for upcoming era support and vote handling, which broadens readiness for future network changes.
* Strengthened **share and serve Leios votes across node flows:** Dingo now handles Leios vote diffusion and vote serving, which improves support for emerging vote traffic.
* Embedded **bundle the Leios testnet configuration:** Dingo now includes the Leios testnet configuration bundle in the release, which makes testnet setup easier and more consistent.
* Broadened **show Conway era protocol settings in epoch responses:** Epoch endpoints now include Conway era protocol parameters, which gives API consumers a clearer view of current network settings.
* Refreshed **improve peer snapshot and topology seeding data:** Dingo now updates snapshot based Genesis seeding and refreshes bundled snapshot and topology data, which improves network bootstrapping and peer selection.

### 🔧 Fixes

* Optimized **reduce deferred index rebuild work after backfills:** Dingo now splits urgent and lazy rebuild work after API backfills, which lowers rebuild cost and helps recovery finish sooner.
* Updated **align the bundled devnet with protocol version 11:** The bundled devnet now uses the newer minimum protocol version, which keeps local testing aligned with current expectations.
* Stabilized **prevent sync stalls close to the chain tip:** Dingo now avoids getting stranded close to the chain tip during sync and bootstrap flows, which keeps nodes moving forward more reliably.

### 🧪 Examples, Docs, and Release Tooling

* Added **show a local Blockfrost explorer example:** The repository now includes an example for running the explorer against a local Blockfrost service, which makes local exploration setups easier to follow.
* Documented **publish the v0.50.2 release notes:** The repository release notes now include the previous version page, which keeps recent release history complete.

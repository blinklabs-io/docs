## v0.51.0 (June 7, 2026)

**Title:** Configurable storage tuning, ledger peer snapshots, and Leios-era groundwork

**Date:** June 7, 2026

**Version:** v0.51.0

Hi folks! Here’s what we shipped in v0.51.0.

### 💪 Improvements

* Improved **configure Badger compression with finer control:** Dingo now lets operators tune compression settings, including ZSTD compression levels, which makes storage behavior easier to tailor to different workloads.
* Expanded **query ledger peer snapshots directly:** Local state queries can now return ledger peer snapshots, which makes it easier to inspect current peer data for network operations.
* Refined **speed up SQLite backfill writes:** Dingo now uses fixed shape database writes during API backfill paths, which improves write efficiency when large backfills run.
* Enhanced **select logging output and verbosity:** Dingo now lets deployments choose text or JSON logs and set log levels, while keeping text as the default unless JSON is configured explicitly.
* Advanced **prepare Dingo for Dijkstra and Leios voting flows:** Dingo now includes early groundwork for upcoming era support and vote handling, which broadens readiness for future network changes.
* Strengthened **serve and diffuse Leios votes across Ouroboros flows:** Ouroboros can now share Leios votes and serve them through LeiosFetch, which improves support for emerging vote traffic.
* Embedded **ship the Leios testnet configuration bundle:** Dingo now includes the Leios testnet configuration bundle in the release, which makes testnet setup easier and more consistent.

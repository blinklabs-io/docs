## v0.51.0 (June 7, 2026)

**Title:** Configurable storage tuning, ledger peer snapshots, and Leios-era groundwork

**Date:** June 7, 2026

**Version:** v0.51.0

Hi folks! Here’s what we shipped in v0.51.0.

### 💪 Improvements

* Improved **configure Badger compression with finer control:** Dingo now lets operators tune compression settings, including ZSTD compression levels, which makes storage behavior easier to tailor to different workloads.
* Expanded **query ledger peer snapshots directly:** Local state queries can now return ledger peer snapshots, which makes it easier to inspect current peer data for network operations.
* Refined **speed up SQLite backfill writes:** Dingo now uses fixed shape database writes during API backfill paths, which improves write efficiency when large backfills run.
* Enhanced **select logging output and verbosity:** Dingo now lets deployments choose text or JSON logs and set log levels, while keeping text as the default unless JSON is configured explicitly.
* Advanced **prepare Dingo for Dijkstra and Leios voting flows:** Dingo now includes early groundwork for upcoming era support and vote handling, which broadens readiness for future network changes.
* Strengthened **serve and diffuse Leios votes across Ouroboros flows:** Ouroboros can now share Leios votes and serve them through LeiosFetch, which improves support for emerging vote traffic.
* Embedded **ship the Leios testnet configuration bundle:** Dingo now includes the Leios testnet configuration bundle in the release, which makes testnet setup easier and more consistent.
* Broadened **expose Conway era protocol settings through epoch endpoints:** Blockfrost epoch responses now include Conway era protocol parameters, which gives API consumers a clearer view of current network settings.
* Refreshed **improve peer snapshot and topology seeding:** Dingo now updates snapshot based Genesis seeding and refreshes bundled snapshot and topology data, which improves network bootstrapping and peer selection.

### 🔧 Fixes

* Optimized **reduce deferred index rebuild work after backfills:** Dingo now splits urgent and lazy rebuild work after API backfills, which lowers rebuild cost and helps data recovery finish sooner.
* Updated **align the bundled devnet with protocol version 11:** The bundled devnet now uses the newer minimum protocol version, which keeps local testing aligned with current expectations.
* Stabilized **prevent near tip sync stalls:** Dingo now avoids getting stranded close to the chain tip during sync and bootstrap flows, which keeps nodes moving forward more reliably.

### 🧪 Examples, Docs, and Release Tooling

* Added **show a local Blockfrost explorer example:** The repository now includes an example for running the explorer against a local Blockfrost service, which makes local exploration setups easier to follow.
* Documented **publish the v0.50.2 release notes:** The repository release notes now include the previous version page, which keeps recent release history complete.

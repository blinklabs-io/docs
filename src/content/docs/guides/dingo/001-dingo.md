A Cardano blockchain data node written in Go which actively participates in network communications on the Cardano blockchain using the Ouroboros Network Node-to-Node family of mini-protocols.  
A Cardano node in Go that now gives operators stronger observability and troubleshooting support across day to day operations.  

⚠️ This is a work in progress and is currently under heavy development

<br>

- Observe inbound peer governance and tune operator controls with more clarity.
- Monitor node overview, block production, peer health, mempool activity, and resource usage with Grafana and Prometheus assets.
- Track era transition state more safely and maintain stronger blockfetch resilience during chain changes.

## How do I actually use Dingo?

This guide covers running Dingo on the Cardano Preview network, from getting the binary and configuration files to bootstrapping from a Mithril snapshot and interacting with the node through the Cardano CLI. It also covers the areas operators now monitor more effectively, including peer health, blockfetch behavior, mempool and resource usage, and era transition state.
